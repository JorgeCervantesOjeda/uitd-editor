# Guía de estilo para documentación

## Objetivo

Mantener documentación clara, consistente y correcta en español.

## Reglas de idioma

- Usa español neutro y técnico.
- Escribe tildes y caracteres especiales correctamente: `á, é, í, ó, ú, ü, ñ`.
- Usa signos de apertura cuando corresponda: `¿` y `¡`.
- Evita mezclar inglés y español en una misma frase, salvo nombres técnicos.

## Terminología recomendada

- `aplicación` en lugar de `app` (excepto en nombres propios).
- `validación`, `configuración`, `producción`, `edición`, `interacción`.
- `importación` y `exportación`.
- `menú`, `diálogo`, `acción`, `condición`, `transición`.

## Formato

- Títulos claros y cortos.
- Párrafos breves.
- Listas con verbos en infinitivo o sustantivos, pero de forma consistente.
- Rutas, comandos y código en monoespaciado (backticks), por ejemplo: `src/validation/diagramValidation.ts`.

## Convenciones para este repositorio

- Documentación principal en `README.md`.
- Detalle técnico en `DOCUMENTACION_APP.md`.
- Mantener este documento como referencia editorial para nuevos `.md`.
